# هندسة النظام - System Architecture

## نظرة عامة
نظام مراكز الأوائل للرعاية النهارية هو تطبيق ويب متكامل مبني على Flask Framework.

## البنية المعمارية

### الطبقات الرئيسية

```
┌─────────────────────────────────────────────┐
│           Presentation Layer                │
│        (HTML/CSS/JavaScript)                │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│          Application Layer                  │
│         (Flask Routes/APIs)                 │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│           Business Logic Layer              │
│         (Services/Validators)               │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│            Data Access Layer                │
│          (SQLAlchemy Models)                │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│             Database Layer                  │
│        (PostgreSQL/SQLite)                  │
└─────────────────────────────────────────────┘
```

## المكونات الرئيسية

### 1. التطبيق الرئيسي
- **الملف**: `ملف موقع الاوائل 2.py`
- **الوظيفة**: نقطة الدخول الرئيسية للتطبيق
- **المسؤوليات**:
  - تهيئة Flask app
  - تسجيل Routes
  - إعداد Extensions

### 2. إدارة الإعدادات
- **الملف**: `config.py`
- **الوظيفة**: إدارة إعدادات التطبيق
- **الميزات**:
  - إعدادات متعددة (Development, Production, Testing)
  - تحميل المتغيرات البيئية
  - إعدادات الأمان

### 3. نماذج قاعدة البيانات
- **الوظيفة**: تعريف هيكل قاعدة البيانات
- **النماذج الرئيسية**:
  - User - المستخدمون
  - Student - الطلاب
  - Teacher - المعلمون
  - Classroom - الفصول
  - Assessment - التقييمات
  - Skill - المهارات

### 4. المصادقة والتفويض
- **التقنية**: JWT (JSON Web Tokens)
- **الميزات**:
  - تسجيل الدخول
  - التسجيل
  - إدارة الصلاحيات (Roles)
  - حماية Endpoints

### 5. معالجة الأخطاء
- **الملف**: `error_handlers.py`
- **الوظيفة**: معالجة الأخطاء بشكل موحد
- **الميزات**:
  - استجابات خطأ موحدة
  - تسجيل الأخطاء
  - رسائل خطأ واضحة بالعربية

### 6. التحقق من البيانات
- **الملف**: `validators.py`
- **الوظيفة**: التحقق من صحة المدخلات
- **التحققات**:
  - البريد الإلكتروني
  - رقم الهوية
  - رقم الجوال
  - التواريخ
  - كلمات المرور

### 7. الأدوات المساعدة
- **الملف**: `utils.py`
- **الوظيفة**: وظائف مساعدة متنوعة
- **الميزات**:
  - رفع الملفات
  - معالجة التواريخ
  - الترقيم (Pagination)
  - التحكم في الوصول

### 8. تسجيل الأحداث
- **الملف**: `logging_config.py`
- **الوظيفة**: تسجيل أحداث النظام
- **أنواع السجلات**:
  - سجلات عامة
  - سجلات الأخطاء
  - سجلات الأمان

### 9. قاعدة البيانات
- **الملف**: `database_helpers.py`
- **الوظيفة**: عمليات قاعدة البيانات
- **الميزات**:
  - تحويل النماذج إلى JSON
  - الإدخال الجماعي
  - البحث والترقيم
  - الحذف الناعم (Soft Delete)

## تدفق البيانات

### 1. طلب API
```
Client → Flask Route → Validator → Business Logic → Database
```

### 2. استجابة API
```
Database → Serializer → JSON Response → Client
```

### 3. المصادقة
```
Client → JWT Token → Verification → Authorization → Resource
```

## الأمان

### طبقات الأمان
1. **طبقة النقل**: HTTPS/TLS
2. **طبقة التطبيق**: JWT, CORS, CSRF Protection
3. **طبقة قاعدة البيانات**: تشفير كلمات المرور, SQL Injection Prevention
4. **طبقة البيانات**: التحقق من المدخلات, Sanitization

### آليات الحماية
- تشفير كلمات المرور بـ bcrypt
- JWT للمصادقة
- حماية CSRF
- التحقق من المدخلات
- Rate Limiting (قيد التطوير)
- تسجيل الأحداث الأمنية

## قابلية التوسع

### التوسع الأفقي (Horizontal Scaling)
- استخدام Load Balancer
- تشغيل نسخ متعددة من التطبيق
- قاعدة بيانات مركزية

### التوسع العمودي (Vertical Scaling)
- زيادة موارد الخادم
- تحسين الاستعلامات
- استخدام Cache

### التخزين المؤقت (Caching)
- Redis للـ Cache (قيد التطوير)
- Cache للاستعلامات الشائعة
- Cache للجلسات

## الأداء

   - استخدام Indexes

   - Connection Pooling

2. **التطبيق**
   - Lazy Loading
   - Pagination
   - Compression

3. **الواجهة الأمامية**

   - CDN للملفات الثابتة
   - Browser Caching

## المراجع التقنية

### التقنيات المستخدمة
- **Backend**: Python 3.8+, Flask 3.0
- **Database**: SQLAlchemy ORM, PostgreSQL/SQLite
- **Authentication**: Flask-JWT-Extended
- **Email**: Flask-Mail
- **Migration**: Flask-Migrate
- **Security**: bcrypt, werkzeug
- **Image Processing**: Pillow
- **Frontend**: HTML5, CSS3, JavaScript
- **Deployment**: Gunicorn, Nginx, Supervisor

### الأنماط المعمارية
- MVC (Model-View-Controller)
- RESTful API
- Repository Pattern
- Dependency Injection

### أفضل الممارسات
- SOLID Principles
- DRY (Don't Repeat Yourself)
- Clean Code
- Security First
- Test-Driven Development (قيد التطوير)

## الخلاصة

النظام مصمم بشكل معياري وقابل للتوسع، مع التركيز على:
- **الأمان**: حماية شاملة على جميع المستويات
- **الأداء**: تحسينات مستمرة
- **الصيانة**: كود نظيف وموثق
- **التوسع**: قابلية النمو المستقبلي
