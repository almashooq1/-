# دليل المساهمة في المشروع

## مرحباً بك!
نشكرك على اهتمامك بالمساهمة في نظام إدارة مراكز الأوائل للرعاية النهارية.

## معايير الكود

### Python
- اتبع معايير PEP 8
- استخدم أسماء متغيرات وصفية باللغة العربية أو الإنجليزية
- أضف تعليقات توضيحية للأجزاء المعقدة
- تأكد من أن الكود نظيف وقابل للقراءة

### Git Commits
- استخدم رسائل commit واضحة ووصفية
- اكتب بالعربية أو الإنجليزية
- ابدأ برسالة موجزة (أقل من 50 حرف)
- أضف تفاصيل إضافية إذا لزم الأمر

مثال:
```
إضافة ميزة تصدير التقارير

- تصدير تقارير الطلاب بصيغة PDF
- تصدير تقارير الحضور بصيغة Excel
- إضافة خيارات تخصيص التقارير
```

## عملية المساهمة

### 1. Fork المشروع
انقر على زر "Fork" في أعلى الصفحة

### 2. Clone المستودع
```bash
git clone https://github.com/your-username/-.git
cd -
```

### 3. إنشاء فرع جديد
```bash
git checkout -b feature/اسم-الميزة
```

### 4. إجراء التغييرات
- اكتب الكود
- اختبر التغييرات
- تأكد من عدم كسر الوظائف الموجودة

### 5. Commit التغييرات
```bash
git add .
git commit -m "وصف التغيير"
```

### 6. Push إلى GitHub
```bash
git push origin feature/اسم-الميزة
```

### 7. إنشاء Pull Request
- افتح Pull Request من فرعك إلى main
- اكتب وصفاً واضحاً للتغييرات
- انتظر المراجعة

## أنواع المساهمات

### إصلاح الأخطاء
- ابحث عن المشاكل الموجودة في Issues
- أو أنشئ Issue جديد لوصف المشكلة
- قم بإصلاح المشكلة في فرع جديد

### ميزات جديدة
- ناقش الميزة المقترحة في Issues أولاً
- تأكد من أنها تتوافق مع أهداف المشروع
- قم بتطوير الميزة في فرع منفصل

### تحسين الوثائق
- تحسين README
- إضافة تعليقات توضيحية
- كتابة أدلة المستخدم

### تحسين الأداء
- تحديد مشاكل الأداء
- اقتراح وتنفيذ حلول
- قياس التحسينات

## الإبلاغ عن المشاكل

عند إنشاء Issue جديد، يرجى تضمين:
- وصف واضح للمشكلة
- خطوات إعادة إنتاج المشكلة
- النتيجة المتوقعة والفعلية
- لقطات الشاشة إن أمكن
- معلومات البيئة (نظام التشغيل، نسخة Python، إلخ)

## التواصل
للأسئلة أو المساعدة، يمكنك:
- فتح Issue للمناقشة
- التواصل عبر البريد الإلكتروني

## شكراً لك!
كل مساهمة، صغيرة كانت أم كبيرة، تساعد في تحسين النظام. نقدر وقتك وجهدك!
